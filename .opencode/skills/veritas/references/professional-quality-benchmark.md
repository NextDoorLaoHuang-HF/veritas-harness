## 关键结构特征（vs 原有简略示例的差距）

### 1. 数据深度

| 特征 | 专业法律研究样板 | 原有示例 |
|------|-----------|---------|
| 费用表行数 | 30-70行（马来西亚法院费用表完整列出） | 无 |
| 对比表 | 机构/规则/费用/特点四列 | 3列笼统对比 |
| 脚注 | ①-⑥6条（含口径说明、计算方式、例外） | 无 |

### 2. 案例摘录深度

| 特征 | 专业法律研究样板 | 原有示例 |
|------|-----------|---------|

| 诉请 | 逐项列出含金额 | 无 |

| 判决主文 | 逐项列出 | 无 |
| 🔴🟡标记 | 有（裁判倾向一目了然） | 无 |

### 3. 实务指引特征

| 特征 | 专业法律研究样板 | 原有示例 |
|------|-----------|---------|
| 痛点引入 | "你是不是也遇到过…" | 无 |
| 金额区间 | "单字判赔1000-5000元" | 无具体数字 |
| **划重点！** | 有（如"单纯的商业性XX侵权，不会被判赔礼道歉"） | 无 |
| 双线策略 | 和解层面/应诉层面并列 | 无 |
| 尾注 | *本文档为外部资料收录…* | 无 |

### 4. 视觉层次标记

专业法律研究样板使用三种 blockquote 标记创建视觉层次：

- `> 📊 **标题**` — 对比/评估摘要（开篇）
- `> 📋 **标题**` — 数据明细表（章节内）
- `**术语定义**` 加粗段落标题 + 详细说明段

## 阈值校准推导

### deep-research 阈值

- **min_lines=80**：样板197行，但含70行费用表。去掉费用表后约127行正文，80行是最低可接受。
- **min_bold=6**：样板有8处加粗（关键推理+实务结论），6处是下限。
- **min_tables=2**：样板有3个表（雷达对比+法院费用+仲裁费用），2个是最低要求。
- **require_footnotes**：样板的6条脚注对数据表理解至关重要，不可省略。

### practical-guide 阈值

- **min_lines=60**：样板86行，60行是去尾注后的最低量。
- **min_bold=6**：样板有7处加粗（划重点+结论判断），6处是下限。
- **require_emphasis**：**划重点！**是专业法律研究样板最显著的读者引导特征，必须存在。

### case-research 阈值

- **min_lines=80**：样板114行（4案例），3案例最低约80行。
- **min_bold=4**：每个案例至少1处关键推理加粗，4案例=4处。
- **min_tables=1**：案例概要表是必须的。
- **🔴🟡标记**：样板用🔴标注不利裁判、🟡标注有利/中立裁判，是读者快速定位的关键。

## 校准方法

当样板更新或新增类型时，用以下步骤重新校准：

1. 读取新样板，统计行数/加粗数/表格数/脚注数
2. 计算正文行数（去掉纯数据行）
3. 取正文行数的60-70%作为 min_lines
4. 取加粗数的75%作为 min_bold
5. 表格数直接取（不可降级）
6. 更新 `finalize.py` 中的 `QUALITY_THRESHOLDS`
7. 更新 SKILL.md 中的阈值表
8. 更新 README.md 中的阈值表
